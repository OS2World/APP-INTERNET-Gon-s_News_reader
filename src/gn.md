


GN(1)              UNIX-OMRON SOFTWARE tool box-Manual           GN(1)



名前
	gn - ニュースリーダ（_Gon's _Newsreader）

形式
	gn [-h hostname] [-d domainname][-lsnV]
		-h hostname	ニュースサーバを指定する *1

		-d domainname	ドメイン名を指定する

		-l		ローカルマシンのニューススプールを見る
			-s と同時には指定できない *2
			DOS/Windows の場合は使用できない

		-s		gnspool と組み合わせて使用する
			-l と同時には指定できない *2

		-n		未読の記事数を表示して終了

		-V		バージョンの表示
				Windows の場合は使用できない



		*1	ローカルマシンのニューススプールを見る場合は、.
			newsrc-NNTPSERVER/.gnrc-NNTPSERVER を探すキー
			ワードとしてのみ使用する
		*2	-l と -s の違いは、ポスト／メイル時に、
			システムの inews/sendmail を使うか、
			gnspool 用のファイルを作成するか、
			である。

特長
	gn は、「ニュースグループ」「サブジェクト」毎に分類する nntp *1
	対応のニュースリーダである。

	初心者に使いやすいように、数字もしくは簡単なコマンドを入力する
	だけで使用可能なように留意している。

	他にはない「サブジェクトモード」という概念を導入することにより、
	同一の話題を追いかけることが容易である。

	また、ニュースサーバとの通信を最小限におさえているため、起動や
	選択がほとんどリアルタイムに実行される。時間がかかりそうな時は、
	「テケテケ」が表示されるため、安心していられる。

	作者が英語を知らないため、メッセージが日本語なのもまた特長であ
	る。:-p

	その他、特長と思われる点を列挙する。
		既に読んでしまった記事を参照することが可能。
		読んだ記事は自動的にマークされる。
		既読マークを消すことが可能。
		元記事をリカーシブに参照することが可能。
		終了することなく新しい記事を見ることが可能。
		記事をパイプに渡すことが可能。
		inews_nntp のような外部コマンドを必要としない。
		ページャとして less を使用。
		signature *2 を付加した形でエディット可能。
		環境変数などでドメイン名などをカスタマイズ可能。

	*1 nntp: NetNews を扱うプロトコル
	*2 signature: 記事の最後に添付する署名

機能説明
	注：以後の説明では、, で区切って複数の「既読指定」をした 
	.newsrc *1 のことを、「GNUS *2 フォーマットの .newsrc 」という
	表現をしているが、これはおそらく正しい表現ではない。

	注：画面例に使用した、
		ニュースグループ
		サブジェクト
		投稿者
	はすべてフィクションである。実在するものではない。

	*1 .newsrc: ニュースグループごとの既読記事番号を管理するファイル
	*2 GNUS: Emacs 上のニュースリーダ

    １．起動
	起動すると以下のようなメッセージが表示される。

	% gn<CR>
		+-----------------------------------------------------------+
		| /home/yamasita/.newsrc から既読情報を読み込んでいます  (1)|
		| GNUS format として扱います                             (2)|
		+-----------------------------------------------------------+
		(2)は .newsrc に , がある場合にのみ 表示される
		+-----------------------------------------------------------+
		| 未読の記事数を数えています                             (3)|
		| ........                                               (4)|
		| 未読の記事が50より多くなっているニュースグループがあります|
		| 50 だけ残して消しますか？(y:消す/その他:全部残す)：    (5)|
		+-----------------------------------------------------------+
		(5)は未読の記事が多い場合にのみ捨てるかどうか聞いてくる。
		捨てる場合は y を入力する。

		その後ニュースグループモードとなる。
	
    ２．モード

	gn には、以下の３つモードがある。

	+----------------------+
	|ニュースグループモード|
	+-------+--------------+
		|
	+-------+--------------+
	|  サブジェクトモード  |
	+-------+--------------+
		|
	+-------+--------------+
	|  アーティクルモード  |
	+----------------------+

	（１）ニュースグループモード
		ニュースグループ一覧と、およその未読記事数を表示するモー
		ドである。

		+-------------------------------------------------------+
		| ニュースグループモード                                |
		|                                                       |
		| 番号 記事数 グループ名                                |
		|    0   152  omronsoft.jokes                           |
		|    1    21  omronsoft.news.group                      |
		|    2    81  omronsoft.rec.autos                       |
		|    3    30  omronsoft.rec.bikes                       |
		| ニュースグループの番号／名前またはコマンド（help=?）：|
		+-------------------------------------------------------+

		未読の記事がないニュースグループや、Unsubscribe（未購
		読）のグループを選択する場合には、ニュースグループの名
		前を直接入力する。
		未読の記事がない場合、スプール上のすべての記事がサブジェ
		クトモードに渡される。
		ここで表示される記事数は正確ではない可能性がある。

	（２）サブジェクトモード。
		（１）で選択したニュースグループのサブジェクト一覧と、
		未読記事数を表示するモードである。

		+-------------------------------------------------------+
		| サブジェクトモード（omronsoft.news.group）            |
		|                                                       |
		| 番号 記事数 サブジェクト                              |
		|    0     3  Re: Prototype Voting omronsoft.jokes      |
		|    1     4  omronsoft.jokes.d as omronsoft.sources.d  |
		|    2     2  Prototype Voting Form omronsoft.living    |
		|    3     2  Creating a newsgroup "omronsoft.living"   |
		|    4     2  why omronsoft.life is better than living  |
		|    5     2  Re: Integration of omronsoft.life.*       |
		|    6     2  Prototype Voting Form omronsoft.jokes.d   |
		|    7     1  Re: omronsoft.jokes.d Hantai!             |
		| サブジェクトの番号またはコマンド（help=?）：          |
		+-------------------------------------------------------+

		Re: *1 がないものはそのサブジェクトの最初の記事が存在
		するもの、Re: があるものはある元記事へのフォローだけが
		あるサブジェクトである。

		*1 Re: 返信のサブジェクトに付加される記号

	（３）アーティクルモード。
		（１）で選択したニュースグループの、（２）で選択したサ
		ブジェクトの投稿者一覧を表示するモードである。
		記事番号／記事の行数／投稿者が表示される。既読の記事で
		あれば、１カラム目に "R" が表示される。
		記事の参照があれば、参照順に並べ変えられ、その親子関係
		が字下げで表される。
		+-------------------------------------------------------+
		| アーティクルモード（omronsoft.news.group:Prototype    |
		|				Voting omronsoft.jokes）|
		|                                                       |
		|   番号  行数 投稿者                                   |
		| R 1308    30 ponpoku@pfrad.omronsoft.co.jp (P Koga)   |
		| R 1340    25  kona@csl.omronsoft.co.jp (S Kona)       |
		|   1328    21   soitoh@icsts1.omronsoft.co.jp          |
		| 記事の番号またはコマンド（help=?）：                  |
		+-------------------------------------------------------+
			少し折り曲げてます。

		既読の記事は、記事番号を入力しないと読めない。


    ３．共通サブコマンド
	各モードのコマンドラインから入力するサブコマンドのうち、共通の
	ものを説明する。

	（１）ポスト	(i:initial post)
		i コマンドで、新規のニュースをポスト（投稿）することが
		できる。

		サブジェクト／アーティクルモードからの場合は、指定した
		時に選択していたニュースグループがデフォルトとして使用
		される。

		エディタ起動時点で、既に $HOME/.signature *1 が付加された
		形になっている。
		ヘッダと、本文の間には、１行以上の空白行が必要である。

		エディタを終了することにより、確認入力の後、ポストされ
		る。ポストされた記事は、$HOME/author_copy に追加される。

		*1 $HOME/.signature: あらかじめ署名のファイルを作成し
		   ておく。$HOME はホームディレクトリをあらわす

	（２）フォローアップ	(f:followup)
		f コマンドで、最後に読んだ記事に対するフォローアップ
		（返信）記事をポストすることができる。

		もしまだ記事を読んでない場合は、実行できない。

		エディタ起動時点で、既に $HOME/.signature が付加された
		形になっている。
		ヘッダと、本文の間には、１行以上の空白行が必要である。

		エディタを終了することにより、確認入力の後、ポストされ
		る。ポストされた記事は、$HOME/author_copy に追加される。

		最後に読んだ記事に、Followup-To: poster *1 の指定があ
		る場合は、確認入力後リプライモードに移行することができ
		る。

		*1 Followup-To: poster
		   Followup-To: はフォローをポストしてほしいニュースグ
		   ループを指定する。フォローでなくリプライメイルがほ
		   しい場合は　Followup-To: poster を指定する事が多い

	（３）リプライ	(r:reply)
		r コマンドで、最後に読んだ記事に対してリプライ（返信）
		メイルを出すことができる。

		もしまだ記事を読んでない場合は、実行できない。

		エディタ起動時点で、既に $HOME/.signature が付加された
		形になっている。
		ヘッダと、本文の間には、１行以上の空白行が必要である。

		エディタを終了することにより、確認入力の後、リプライメ
		イルが送られる。

	（４）キャンセル	(C:cancel)
		C コマンドで、最後に読んだ記事をキャンセル *1 すること
		ができる。
		もしまだ記事を読んでない場合は、実行できない。

		キャンセルできるのは、自分の記事だけである。

		*1 キャンセル：NetNews にポストした記事を取り消すこと。

	（５）保存	（s:save）
		s コマンドで、最後に読んだ記事をファイルに保存すること
		ができる。ファイルが既に存在する場合は、そのファイルに
		追加される。
		保存したファイルは Unix mail 形式であるため、
		Mail -f file として参照することができる。

		もしまだ記事を読んでない場合は、実行できない。

		コマンドラインで、s file のようにファイル名を指定する
		ことも可能である。file の指定には、環境変数（$HOME な
		ど）や、~/ *1 を使用することができる。
		指定されたファイル名を展開した結果が、/ から始まらない
		場合は、$SAVEDIR *2 からの相対とみなす。

		*1 ~/: ホームディレクトリをあらわす。$HOME/ と同値
		*2 $SAVEDIR: コンパイル時や gnrc で指定した記事を保存
		   するためのディレクトリ

	（６）リファレンス	(R:reference)
		R コマンドで、最後に読んだ記事の元記事 *1 を表示するこ
		とができる。

		もしまだ記事を読んでない場合は、実行できない。

		元記事を探せるのは、当然 References: ヘッダを持った記
		事だけである。References: ヘッダが途中で切れた記事を時
		おり見かける。このような記事の元記事は探せない。

		cancel , expire *2 などにより元記事が存在しない場合も
		ある。


		*1 元記事：今読んでいる記事が返信であれば、その返信元
		   の記事
		*2 expire: ニュースサーバから、古い記事を消すこと。

	（７）メッセージＩＤ *1
		指定したメッセージＩＤの記事を表示する。
		入力は、マウスでのカットアンドペーストを推奨する。

		*1 メッセージＩＤ: 各記事がもつ固有の文字列。
		   Message-ID: というヘッダにあり < で始まり > で終わ
		   る。

	（８）パイプ	(|:pipe)
		| コマンドで、unix コマンドの標準入力へ、最後に読んだ
		記事を渡すことができる。

		もしまだ記事を読んでない場合は、実行できない。

		コマンドラインで |<CR> と入力すると、渡すコマンド名の
		入力となる。 コマンドラインで、| grep omron のように指
		定することも可能である。
		
	（９）キル	(K:kill)
		最後に読んだ記事と同じ From、同じ Subject の記事を既読
		とするよう登録する。
		SPAM のようにいくつものニュースグループに同じ From で
		同じ Subject の記事がポストされている際、K コマンドを
		実行しておくと、他のニュースグループに移った際にその記
		事を既読とする。
		登録を保存したり、ファイルから読んでくることはできない

	（１０）前画面	(p:previous)
		ニュースグループ、サブジェクト、投稿者の一覧が画面に収
		まらない場合、p コマンドで前の画面を表示できる。
		今表示されているのが、最初の画面である場合は、最初の画
		面を再表示する。
		p 5 のように数を引数として付加した場合はその行数分上か
		ら表示する。

	（１１）次画面	(n:next)
		ニュースグループ、サブジェクト、投稿者の一覧が画面に収
		まらない場合、n コマンドで次の画面を表示できる。
		今表示されているのが、最後の画面である場合は、最後の画
		面を再表示する。
		n 5 のように数を引数として付加した場合はその行数分下か
		ら表示する。

	（１２）メイル	(m:mail)
		m コマンドでメイルを送信することができる。

	（１３）検索	(/,\)
		各モードでリストされる文字列の中から指定した文字列を検
		索する。初回以外は前回指定した文字列が使用される。
		/ string とすると、下方向に、\ string とすると上方向に
		検索する。

	（１４）シェル	(!:shell)
		! コマンドで、unix のコマンドを実行することができる。

		コマンドラインで !<CR> と入力すると、実行するコマンド
		名の入力となる。 コマンドラインで、! ls のように指定す
		ることも可能である。

	（１５）終了	(q:quit)
		現在のモードの一つ上のモードへ戻る。ニュースグループモー
		ドの場合は、（必要ならば .newsrc を更新して）終了する。

	（１６）ヘルプ	(?:hatena)
		現在のモードのコマンド一覧を表示する。モードによって内
		容が異なる。
		ヘルプの”リターンキーを押して下さい：”というプロンプ
		トからそのモードのコマンド（リターンのみを除く）を入力
		することもできる。


    ４．ニュースグループモードのサブコマンド
	コマンド一覧を示す。

	<CR>のみ      :最初のグループを選択する。
	数字<CR>      :指定した番号のグループを選択する。
	名前<CR>      :指定した名前のグループを選択する。
	p [数]<CR>    :前画面（もしくは指定した数だけ上）を表示する。
	n [数]<CR>    :次画面（もしくは指定した数だけ下）を表示する。
	h<CR>         :新しい記事をチェックする。
	l [NG]<CR>    :表示モードを切り替える。
	i<CR>         :記事をポストする。
	R<CR>         :最後に読んだ記事の元記事を表示する。
	f<CR>         :最後に読んだ記事にフォロー記事を出す。
	r<CR>         :最後に読んだ記事にリプライメイルを出す。
	C<CR>         :最後に読んだ記事をキャンセルする。
	s [file]<CR>  :最後に読んだ記事をファイルに保存する。
	| [cmd]<CR>   :最後に読んだ記事をコマンドに渡す。
	m<CR>         :メイルを送信する。
	u [NG]<CR>    :指定したグループはもう読まない。
	S [NG]<CR>    :指定したグループはやっぱり読む。
	c [NG]<CR>    :指定したグループの記事を全部読んだことにする。
	/ [文字列]<CR>:文字列によるニュースグループ名検索。
	\ [文字列]<CR>:文字列によるニュースグループ名逆検索。
	! [cmd]<CR>   :Unix コマンドを実行する。
	w<CR>         :既読情報を .newsrc へ保存する。
	q<CR>         :終了。
	x<CR>         :.newsrc を更新せず終了。
	?<CR>         :ヘルプ。

		[file] はファイル名（省略可能）
		[NG] はニュースグループの番号または名前（省略可能）
		[cmd] は Unix コマンド（省略可能）

	「<CR>のみ」「数字<CR>」「名前<CR>」のいづれかのコマンドで、サ
	ブジェクトモードに移行する。この際、そのニュースグループに記事
	が多い（未読記事数とは必ずしも一致しない）場合、
	読む範囲を指定して下さい（デフォルト：全部(4211-4628),q:読まない）：
	のように聞いてくる。読む範囲を、
		数字		（例：4211）
		-数字		（例：-4300）
		数字-		（例：4600-）
		数字-数字	（例：4300-4400）
		#数字		（例：#100）
	のいづれかの方法で指定する。
	最後の「#数字」は古い順に「数字」個の記事を読む。上記例だと、
	4211-4310 と #100 は同じ意味である。
	なお、<CR> のみは全部を意味する。

	（１）名前
		ニュースグループの名前を直接指定することができる。
		グループが存在するならば購読していないニュースグループ
		であっても、未読の記事がなくても構わない。
		もし未読の記事があれば未読の記事だけを、未読の記事がな
		ければ、そのグループのすべての記事をサブジェクトモード
		に渡す。
		指定された名前のニュースグループがない場合は、指定され
		たニュースグループ名に近い（と gn が思った）ニュースグ
		ループ名を表示し、番号で選択することができる。

	（２）新しい記事をチェック	(h:hash)
		gn 起動後にポストされた記事もリストする。自分自身が
		ポストした記事も、再起動なしに確認することができる。

	（３）表示モードを切り替える	(l:list)
		通常の「購読していて未読記事があるグループを表示する」
		未読グループ表示モードと
		「未読記事のない／購読していないグループも表示する」
		全グループ表示モードとを切替える。
		全グループ表示モードの際、購読していないグループは U が
		表示される。
		引数としてニュースグループのカテゴリを与えるとそのカテ
		ゴリだけを表示する。

		未読グループ表示モード <- l[CR] -> 全グループ表示モード
		    　     |  ^　　　　　　　　　             |  ^
		l comp[CR] |  | l[CR]　　　　　    l comp[CR] |  | l[CR]
		           |  |　　　　　　　　　　           |  |
		           V  |　　　　　　　　　　           V  |
		    comp だけを表示　　　　　　　    　comp だけを表示

	（４）もう読まない	(u:unsubscribe)
		指定したニュースグループをもう読まないようにする。
		デフォルト状態でリストアップされないだけで、l コマンド
		で全グループの表示を行なったり、ニュースグループの名前
		指定をすると読むことも可能である。
		ニュースグループは番号または名前で指定する。コマンドの
		引数として、ニュースグループを与えなかった場合は、ニュー
		スグループの入力となる。
		番号で指定する場合は、11-20 のように範囲を与え、一括し
		て読まなくすることもできる。

	（５）やっぱり読む	(S:subscribe)
		いったん unsubscribe にしてしまったニュースグループを
		subscribe にする。
		ニュースグループは番号または名前で指定する。コマンドの
		引数として、ニュースグループを与えなかった場合は、ニュー
		スグループの入力となる。
		番号で指定する場合は、11-20 のように範囲を与え、一括し
		て読むようにすることもできる。

	（６）全部読んだことにする	(c:catch up)
		指定したニュースグループの全記事に対して、既読のマーク
		をつける。
		引数として、ニュースグループを与えなかった場合は、ニュー
		スグループの入力となる。
		番号で指定する場合は、11-20 のように範囲を与え、一括し
		て読んだことにすることもできる。



     ５．サブジェクトモードのサブコマンド
	コマンド一覧を示す。

	<CR>のみ      :最初の未読のあるサブジェクトを選択する。
	数字<CR>      :指定した番号のサブジェクトを選択する。
	a<CR>         :未読の全記事を選択する。
	p [数]<CR>    :前画面（もしくは指定した数だけ上）を表示する。
	n [数]<CR>    :次画面（もしくは指定した数だけ下）を表示する。
	t<CR>         :長い行を折り曲げて表示する／しない。（トグル）
	P<CR>         :前のニュースグループへ移動する。
	N<CR>         :次のニュースグループへ移動する。
	i<CR>         :現在のグループへ記事をポストする。
	R<CR>         :最後に読んだ記事の元記事を表示する。
	f<CR>         :最後に読んだ記事にフォロー記事を出す。
	r<CR>         :最後に読んだ記事にリプライメイルを出す。
	C<CR>         :最後に読んだ記事をキャンセルする。
	s [file]<CR>  :最後に読んだ記事をファイルに保存する。
	| [cmd]<CR>   :最後に読んだ記事をコマンドに渡す。
	m<CR>         :メイルを送信する。
	u<CR>         :このグループはもう読まない。
	S<CR>         :このグループはやっぱり読む。
	c [範囲]<CR>  :未読の記事を全部読んだことにする。
	l<CR>         :全部の記事を表示する。
	/ [文字列]<CR>:文字列によるサブジェクト検索。
	\ [文字列]<CR>:文字列によるサブジェクト逆検索。
	! [cmd]<CR>   :Unix コマンドを実行する。
	q<CR>         :ニュースグループモードへ戻る。
	?<CR>         :ヘルプ。

		[file] はファイル名（省略可能）
		[cmd] は Unix コマンド（省略可能）

	「<CR>のみ」「数字<CR>」のいづれかのコマンドで、アーティクルモー
	ドに移行する。

	（１）もう読まない	(u:unsubscribe)
		現在表示されているニュースグループをもう読まないように
		する。デフォルト状態でリストアップされないだけで、l コ
		マンドで全グループの表示を行なったり、ニュースグループ
		の名前指定をすると読むことも可能である。

	（２）やっぱり読む	(S:subscribe)
		いったん unsubscribe にしてしまったニュースグループを
		subscribe にする。

	（３）未読の全記事を選択する	(a:all article)
		未読の全記事を順に選択する。
		１０記事読む毎に確認入力を行ない、n を入力することによ
		り途中で中止することができる。

	（４）全部読んだことにする	(c:catch up)
		現在表示されているニュースグループの全記事（もしくは指定し
		た範囲の記事）に対して、既読のマークをつける。

	（５）全部の記事を表示する	 (l:list all)
		現在表示されているニュースグループの全部の記事をリスト
		アップする。
		記事が多い場合は、読む範囲を聞いてくる。指定方法はニュー
		スグループモードからサブジェクトモードに移行する時と同
		じ。

使用上の留意点
	初めて起動する時、junk と control, to.* は デフォルトで 
	Unsubscribe になる。

	複数のサーバにアクセスするばあいは、作成された .newsrc を 
	.newsrc-$NNTPSERVER にリネーム（mv）しておくこと。異なるサーバ
	の .newsrc を使用した場合は、元の情報が失われる。もし同じ名前
	のニュースグループがあった場合は、未読／既読の判断が狂う。

動作指定
	gn の動作を規定する方法を、優先度が高い順に記述すると、
		(1) 初期化ファイル（$HOME/.gnrc-$NNTPSERVER）
		(2) コマンドラインオプション
		(3) 環境変数
		(4) 初期化ファイル（$HOME/.gnrc）
		(5) コンパイル時に指定された値
	がある。
	注：	この優先順位には異論もあろうが -h オプションで切替える
		サーバ毎に .gnrc を持ちたかったのでこのようになった。

初期化ファイル
	$HOME/.gnrc と $HOME/.gnrc-$NNTPSERVER は gn の動作を規定する
	ファイルである。$HOME/.gnrc は全部もしくは大多数のニュースサー
	バに関する設定を、$HOME/.gnrc-$NNTPSERVER にはそのニュースサー
	バにだけ関する設定を記述する。ニュースサーバが１台しかない場合
	は .gnrc  だけで十分である :-p。以後 gnrc と総称する。

	gnrc の書式は各行毎に
	keyword	value
	とし、keyword と value の間は１個以上のホワイトスペースで分割
	する。
	行頭が # の場合は、コメントとみなしその行を無視する。コメント
	は行頭に # がある場合その１行全部をコメントとする。すなわち、
	行末にコメントを書くことはできない。
	ORGANIZATION	OMRON SOFTWARE Co., Ltd.	#組織名
	と書くと、ORGANIZATION という keyword にvalue として
	『OMRON SOFTWARE Co., Ltd.	#組織名』を設定したことになる。
	
	gnrc は１行５１２バイトまで。

	・keyword
		キーワードは英大文字で記述する。

	・value
		各 keyword に設定する値。文字列もしくは数を指定する。
		「ＸＸするかどうか」というフラグには、
			しない:	0,NO,OFF のいづれか
			する:	1,YES,ON のいづれか
		を指定する。
		value は複数行にわたることはできない X-<。
		value の先頭にホワイトスペースを使うことはできない。

	以下に、各キーワードの詳細を記述する。
	説明を読んでも良くわからない項目は、大抵の場合省略してもよい :-)

	カスタマイズには必須の物からまったく趣味の世界の物まで、必要度
	に応じて以下のマークをつける（山下の独断と偏見）
	(A): 必須。
	     コンパイル時に指定されているときは、省略も可能である。コ
	     ンパイルした人に確認すること。
	     よそからバイナリで配布を受けた場合は、必ず指定すること。

	(B): 山下お勧め。B(1) なら１にすることをお勧めする
	(C): 趣味の世界。必要に応じて指定する
	(D): ほとんどつける必要はない
	(E): 書かないことをお勧めする

		デフォルト：keyword-organization

    (2) ニュースシステムに関する設定

	NNTPSERVER	ニュースサーバ(A)
			ローカルスプールを見る場合
				LOCAL_SPOOL	ON
				-l option
				-s option
			は、.newsrc-NNTPSERVER/.gnrc-NNTPSERVER を探す
			キーワードとしてのみ使用する

			nntpserver:8119 のように「ホスト名：ポート番号」
			を指定することもできる。この指定は下の 
			NNTPPORT での指定よりも優先する。

		デフォルト：keyword-nntpserver

	NNTPPORT	NNTP のポート番号
			標準以外のポート番号で動作しているニュースサー
			バと接続する場合は NNTPPORT にそのポートを指定
			する。

			ポートの指定は、gnrc や newsrc のファイル名の
			指定に影響を与えない。
			例:ニュースサーバに nntpserver:8119 を指定して
			も、.newsrc-nntpserver:8119 というファイルは使
			わず、従来通り .newsrc-nntpserver を使用する。

		デフォルト：119 もしくは /etc/services 等の設定

	MODE_READER	Mode reader を送るか否か(D)
		value として 1 を指定すると、NNTP の接続時に
		"MODE reader" コマンドを送信する。
		ニュースサーバが、Inn でかつ転送可能になっているホスト
		上で gn を使用する場合に 1 にする。
		デフォルト：0（Mode reader を送らない）

	NNTP_AUTH	接続時にユーザ認証をおこなうか否か(D)
		value として 1 を指定すると、NNTP の接続時に
		ユーザ認証をおこなう
		デフォルト：0（NNTP の接続時にユーザ認証をおこなわない）

	LOCAL_SPOOL	ローカルマシンのニュースサーバを見るか否か(D)
		value として 1 を指定すると、NNTP を使用せず、直接 
		/usr/{spool|lib}/news を見るようになる。
		gnspool は無視し必ず NNTP を使用する。
		DOS 系では使用できない。
		デフォルト：0（NNTP を使用する）

	USE_HISTORY	history を使うかどうか(C)
		gnspool と組み合わせて使用する場合、value として 1 を
		指定すると、Message-ID による記事表示や、元記事を表示
		できるようになる。
		ただし、gnspool の実行に時間がかかるようになる。
		また、gnspool で取り寄せた記事はニュースサーバにある記
		事のごく一部であるため、指定した記事がある可能性は低い。
		デフォルト：0（history を使わない）
			    ただし -l を指定した時は 1

	POSTPROC	-l 時、ポストする時のプログラム(D)
		LOCAL_SPOOL の場合、ポストする時に使用するプログラムを
		指定する。
		デフォルト：keyword-newslib/inews -h

	NEWSLIB	active のあるディレクトリ(D)
		LOCAL_SPOOL の場合と gnspool と組み合わせて使用する場
		合、ローカルマシンの active のあるディレクトリを指定す
		る。
		デフォルト：keyword-newslib

	NEWSSPOOL	スプールディレクトリ(D)
		LOCAL_SPOOL の場合と gnspool と組み合わせて使用する場
		合、ローカルマシンのニューススプールディレクトリを指定
		する。
		デフォルト：keyword-newsspool

	DOSFS	newsspool として FAT を使うか(D)
		NEWSSPOOL として長いディレクトリ名が使えるファイルシス
		テム（HPFS/NTFS/VFAT）を使用する場合、value として 0
		を指定する。
		1 を指定すると、FAT のファイル名規則から、以下の変換を
		おこなう。
		a) ニュースグループ名の . と . の間が８文字以上の場合、
		   ８文字に切り詰める
			例：fj.fleamarket.misc -> fj\fleamark\misc
		b) 記事番号が８桁を超える場合、８文字目と９文字目の間
		   に . を挿入する
			例：1234567890 -> 12345678.90
		c) ディレクトリ名として + が使えないため、ニュースグルー
		   プ名の + を　` に変更する
			例：fj.lang.c++ -> fj\lang\c``


		DOS 系での初期値は 1（長いファイル名が使えない）
			OS/2(HPFS), WindowsNT(NTFS)/95(V-FAT)上に
			newsspool を作成し、GNUS などと組合わせて使用
			する場合には 0 を指定すると有効。
			ただし、VC++ 4.0 など、長いファイル名が使える
			ようになるコンパイラでコンパイルした場合のみ

		UNIX 系での初期値は 0（長いファイル名が使える）
			newsspool を DOS 系と共有する場合に 1 を指定す
			ると有効

		＃以前存在した DOS_NEWFS とは ON/OFF が逆なので注意！

	HOST	ＰＣのホスト名（DOS 系の場合）(A:DOS)

	LINES	画面行数(D)
		デフォルト：keyword-lines

	COLUMNS	画面桁数(D)

	PROCESS_LANG	ページャ／エディタの漢字コード(D)

		デフォルト：keyword-file_kanji_code

	DISPLAY_LANG	ディスプレイの漢字コード(D)
		デフォルト：keyword-display_kanji_code

	PAGER	ページャ(C)
		デフォルト：keyword-pager

	RETURN_AFTER_PAGER	記事表示後の Return 入力要求の有無(D)
		最終ページで止まらないページャの場合 value として１を
		指定するとリターンキー入力待ちとなる。
		デフォルト：keyword-return_after_pager

	EDITOR	エディタ(C)
		デフォルト：keyword-editor

	USE_SPACE	スペースで読み進めるかどうか(B(1):DOS/C:Unix)
		value として 1 を指定した場合、
		各モードのコマンド入力時にスペースキーを押すと、
		CR を押したのと同様に動作する。
		スペースキーだけで読み進めることができる。
		ただし、正常に動作しない場合がある。
		デフォルト：0

	POWER_SAVE:	ディスクへのアクセスを一時期にするかどうか(D)
		ＨＤへのアクセスを短時間かつ一時期にかためておこなうこ
		とにより、記事を読んでいる間はＨＤのモータをＯＦＦにで
		き、電池を節約することが可能とする。
		ポータブルＰＣをオフラインで使用する場合に有効。

		value として 1 を指定した場合、
		選択したニュースグループの記事を一時的に TMPDIR にコピー
		し、読む時は TMPDIR をアクセスする。
		TMPDIR に RAMDISK を用いることにより、ＨＤ へのアクセ
		スを全く行わなくすることができる。
		HDD へのアクセスを確実になくすことができるが、RAM DISK 
		のために常時メモリを確保しなければ効果がない。

		value として 2 を指定した場合、
		選択したニュースグループの記事をすべて読み出す。
		実際に読む時は通常と同じくディスク上から読み出すが、
		ディスクキャッシュが効いている場合、アクセスを抑えるこ
		とが可能となる。
		TMPDIR には、小さな RAMDISK でよく、多くのメモリをディ
		スクキャッシュ、アプリケーションに割り当てることが可能
		になる。
		デフォルト：0

		複数のサーバにアクセスするばあいは、作成された .newsrc

		異なるサーバの .newsrc を使用した場合は、元の情報が失
		われる。もし同じ名前のニュースグループがあった場合は、
		未読／既読の判断が狂う。
		デフォルト：newsrc（ＤＯＳ系）/.newsrc（Ｕｎｉｘ系）

	SAVEDIR	記事を保存するディレクトリ(C)
		デフォルト：keyword-savedir

	SIGNATURE	シグネーチャのファイル(D)
		もし '/' から始まらない場合は、$HOME からの相対パスと
		みなす。
		デフォルト：keyword-signature

	AUTHOR_COPY	ポストした記事を保管するファイル(C)
		もし '/' から始まらない場合は、$HOME からの相対パスと
		みなす。
		Unix mail 形式であるため、Mail -f author_copy で参照可能。
		必要ない場合は、/dev/null を指定する。
		デフォルト：keyword-authorcopy

	TMPDIR	テンポラリファイルを作成するディレクトリ(D)
	TMP	テンポラリファイルを作成するディレクトリ(C)
	TEMP	テンポラリファイルを作成するディレクトリ(D)
		テンポラリファイルを作成するのは、以下の場合である。
		(1) 記事を読む場合
			-> 記事をいったんテンポラリファイルに書き、
			   ページャに渡す
			   ただし、ページャに直接記事を渡せる場合 *1 は、
			   テンポラリファイルを作成しない

			   *1 以下の全ての条件を満たす場合
			   ・ローカルスプールを見る
			   ・MIME decode しない
			   ・ローカルスプールとページャの
			     漢字コードが同じ

		(2) ポスト／フォロー記事、リプライメイルを書く場合
			-> テンプレートをテンポラリファイルに書き、
			   エディタに渡す
		デフォルト：keyword-tmpdir

	JNAMESFILE	docs.jp/jnames 参照(D)
		デフォルト：なし

    (7) ニュースグループに関する設定

	GNUS_FORMAT	GNUS format の .newsrc を扱うかどうか(D)
		value として 1 を指定した場合
		 , で区切って複数の「既読指定」をすることが可能となる。
		value として 0 を指定した場合各グループ毎に既読が連続
		する最後の番号だけを .newsrc に保存する。

		0 を指定すると .newsrc に保存できるのは、ニュースグルー
		プ単位に連続して読んだ最後の記事番号だけである。よって、
		再起動すると、読んだはずの記事が未読になっている場合が
		ある。
		例	a というサブジェクトに、1 2 4 5 の記事があり、
			全部読んだとする。
			b というサブジェクトに、3 の記事があるが、これ
			は読んでいないとする。
			この状態で終了し再起動すると 1 2 は既読になる
			が 3 4 5は未読のままである。
		 , で区切って複数の「既読指定」をすることが不可能なニュー
		 スリーダを使わないなら、0 は指定しないことを推奨する。

		ただし 0 を指定しても、読み込んだ .newsrc 内に、
			"," がある
			1 以外から始まる
		行がある場合、自動的に GNUS format モードになる。
		デフォルト：1

	NG_SORT		新しくできたニュースグループのリスト位置(B(1))
		newsrc が無かった場合
			value として 0 を指定した場合
				ニュースサーバが返した順にリスト
			value として 1 or 2 を指定した場合
				昇順にソートしてリスト

		newsrc が有った場合
			value として 0 を指定した場合
				既にあるリストの末尾に
				ニュースサーバが返した順に追加
			value として 1 を指定した場合
				昇順にソートしてリストに追加
				ただし、newsrc が昇順になっていなかっ
				た場合、昇順にならない場合がある。
			value として 2 を指定した場合
				カテゴリ内で昇順にソートしてリストに追
				加
				ただし、newsrc がカテゴリ毎になってい
				なかった場合、カテゴリ内で昇順になって
				いなかった場合、昇順にならない場合がある。
		デフォルト：0

	UNSUBSCRIBE	 購読しないニュースグループ(C)
		全く読まないニュースグループ群を , で区切って列挙する。
		指定されたニュースグループ群はニュースサーバに存在しな
		いものとして扱われる。
		例：	UNSUBSCRIBE alt.a, bionet
			alt.a で始まるニュースグループと 
			bionet で始まるニュースグループは読まない。

		正規表現もどきは以下を使うことができる
		・[] を使った範囲指定
			例：	UNSUBSCRIBE alt.[a-s]
				alt.の次の文字が a から s の
				ニュースグループは読まない
		・! による否定
			例：	UNSUBSCRIBE alt.,!alt.sources
				alt は読まないが alt.sources だけは読む
		・全てにマッチする all

		できるだけたくさんのニュースグループ群を指定することに
		よりメモリを節約することが可能になる。
		デフォルト：なし

	SUBSCRIBE	 購読するニュースグループ(D)
		UNSUBSCRIBE の逆で、購読するグループを列挙する。
		UNSUBSCRIBE の指定があると、SUBSCRIBE の指定は無視する。
		
			！！！警告！！！
		SUBSCRIBE に指定のないグループが作成されても、作成され
		たことを知る方法がない。
		よって、このキーワードを使用することはお勧めしない。
			！！！警告！！！
		デフォルト：なし

	ARTICLE_LIMIT	起動時記事が多い時に確認入力するしきい値(D)
		起動時、ARTICLE_LIMIT よりも既読の記事が多い場合、消す
		かどうかの確認入力を行なう。
		0 を指定するとこのチェックはおこなわない。
		デフォルト：50

	ARTICLE_LEAVE	起動時、多い記事を消す時に残す数(D)
		デフォルト：50

	SELECT_LIMIT	選択時記事が多い時に確認入力するしきい値(D)
		ニュースグループ選択時、記事の数が指定された数より多い
		場合、読む範囲の指定を行なう。
		0 を指定するとこのチェックはおこなわない。
		ただし、大量の記事を取り寄せるとメモリを大量に消費する
		ので、５０程度を指定することを推奨する。
		デフォルト：50

	SELECT_NUMBER	選択時記事が多い時の確認入力時のデフォルト記事数(D)
		上記 SELECT_LIMIT で確認入力をおこなうときのデフォルト
		の記事数を指定する。
		数字を指定すると、未読の最初の記事から指定した数の記事
		を取り寄せる。
		0 を指定するとデフォルトは全記事を取り寄せる。
		大量の記事を取り寄せるとメモリを大量に消費するので、５
		０程度を指定することを推奨する。
		デフォルト：0

    (8) 読み方に関する設定
	DISP_RE		サブジェクトモードで、Re: を表示するかどうか(C)
		value として 0 を指定すると、サブジェクトモードで、全
		ての記事がフォローの場合も Re: を表示しない。
		デフォルト：0

	CLS		各モードで画面クリアをおこなうか否か(C)
		value として 1 を指定すると、各モードのリスト前に画面
		をクリアする。スクロールが遅い環境では有効かも知れない。
		ただし、画面クリアシーケンスはソースにハードコートされ
		ていて、端末毎に変更することはできない。X-<
		また、ESC [ H ESC [ 2 J を処理できない端末では、正常に
		動作しない。
			例：ANSI.SYS を組み込んでいない DOS/V
		デフォルト：0

	DESCRIPTION	 各グループの説明を表示するかどうか(D)
		value として 1 を指定した場合、ニュースグループモード
		で各グループの説明を表示する。
		ニュースサーバによってはサポートしていない場合、サポー
		トしていても説明文がないグループがある場合がある。
		デフォルト：0

	KILL_CONTROL	コントロールメッセージを既読とするかどうか(D)
		デフォルトで既読とするコントロールメッセージを指定する。
		各ビット毎に既読とするコントロールメッセージが割り当て
		てあり、対応するコントロールメッセージのみが既読となる。
			1: cancel を既読とする
			2: ihave を既読とする
			4: sendme を既読とする
			8: newgroup を既読とする
			16: rmgroup を既読とする
			32: checkgroups を既読とする
			64: sendsys を既読とする
			128: version を既読とする
			256: control: ヘッダのない記事を既読とする

		以下のビットは多少意味が異なる
			512: ニュースグループ control を選択した時には
			     Subject の変わりに Control を表示する。
			1024: ニュースグループ junk を選択した時には
			      Subject の変わりに Newsgroups を表示する。

		例：	gnrc に
			KILL_CONTROL	35
			と記述すると、
			cancel,ihave,checkgroups を既読とする
		デフォルト：0


	AUTOLIST	自動表示するかどうか(B(0))
		value  として 0 を指定すると画面表示が最低限だけになる。
		使い慣れた者が遅い回線を使用する場合に指定すると良い。
		初心者には勧められない。
		Windows 版は必ず自動表示する。
		デフォルト：1

	SHOW_INDENT	モード毎に字下げするカラム数(C)
		value として、正の整数を与えると、
			サブジェクトモードではその数分
			アーティクルモードではその数の２倍分
		字下げして表示する。
		デフォルト：0

	SHOW_TRUNCATE	長い行を折り曲げて表示するかどうか(B(0))
		サブジェクトモード、アーティクルモードの表示が１行に収
		まらない場合、
		value として 1 を指定すると折り曲げて次の行に表示する。
		スクロールアップして上の方が見えない場合がある。
		value として 0 を指定すると１行に収まるよう右側を切り
		捨てる。切り捨てられた部分だけが異なるサブジェクトが同
		一のサブジェクトに見える場合がある。
		デフォルト：1
		
	MIME_HEAD	ヘッダ部分に MIME を使用するかどうか(C)
		value として 1 を指定すると、ヘッダ部分に MIME を使用
		することが可能になる。
		また、その他の value として
			2 を指定するとデコードのみ
			4 を指定するとエンコードのみ
		も使用可能である。
		6 や 7 を指定すると 1 と同様デコード、エンコード両方を
		おこなう
		ただし、このキーワードが使えるのは、MIME を使用するよ
		うにコンパイルされている場合のみに限る。
		デフォルト：0

	GMT2JST		Date: ヘッダを JST で表示するかどうか(C)
		value として 1 を指定すると、Date: ヘッダを JST で表示
		する。記事にもともとついている Date: ヘッダが
		[day, ]dd mm yy GMT という format 以外は無視する。
		デフォルト：0

    (9) 書き方に関する設定

	NSUBJECT	日本語 Subject を使用するかどうか(C)
		value として 1 を指定した場合、日本語サブジェクトを使
		うことを可能とする。

		この指定がされると、
		ポスト時
			日本語サブジェクトを聞き X-Nsubject: ヘッダに
			する。

		フォロー時
			元記事に X-Nsubject: があった場合は、必要に応
			じて Re: を付加し、X-Nsubject:  ヘッダにする。
		デフォルト：0

	RN_LIKE		rn ライクな followup や reply(C)
		value として 1 を指定すると、followup 時に
		「From article <message> by whom」形式ではなく
		「In article <message> who writes:」形式になり、
		reply 時にも同様の形式で引用を示すタグが付く。
		ニュースリーダ rn を意識したもの。
		デフォルト：0

	CC2ME		リプライメールを自分にもカーボンコピー(C)
		value として 1 を指定すると、リプライメールを自分にも
		Cc: するようになる。
		value として 2 を指定すると、リプライメールを自分にも
		Bcc: するようになる。
		デフォルト：0

	SUBSTITUTE_HEADER	送信時に From: などを書き換えるか(C)
		value として 1 を指定すると、送信する記事／メイルに
		From: などの発信者情報があった場合に、それを gn の設定
		に従ったものと置き換える
		デフォルト：1


    (10) その他の設定

	WITH_GNSPOOL	gnspool と組み合わせて使用するか否か(D)
		value として 1 を指定すると、-s オプションを指定した場
		合と同じように gnspool と組み合わせて使用するモードと
		なる。
		デフォルト：0

	GNSPOOL_LANG	gnspool が作成するスプールの漢字コード(C)
		JIS,EUC,SJIS を設定する
		デフォルト：JIS

    (11) gnspool にだけ有効な設定
　　　
	FAST_CONNECT	gnspool が active を取得するか否か(C)
		value として 1 を指定すると、gnspool に -f オプション
		を指定した場合と同じように起動が高速になる。
		初回の起動時は FAST_CONNECT 1 を指定してはいけない。
		デフォルト：0（active を取得する）

	REMOVE_CANCELED	gnspool がキャンセルされた記事を消すか否か(D)
		value として 1 を指定すると、gnspool が記事取得時に記
		事がキャンセルされていないかをチェックし、もしキャンセ
		ルされていて、かつローカルスプールに記事ファイルがあれ
		ばそのファイルを消す。
		処理速度は遅くなる。
		デフォルト：0（キャンセルされた記事を消さない）

	GNSPOOL_MIME_HEAD	gnspool が MIME を使用するかどうか(C)
		value として 1 を指定すると、gnspool が記事を取寄せる
		際にヘッダ部分に MIME デコードを行う。また、記事／メイ
		ルを送信する際、ヘッダ部分をエンコードする
		また、その他の value として
			2 を指定するとデコードのみ
			4 を指定するとエンコードのみ
		も使用可能である。
		6 や 7 を指定すると 1 と同様デコード、エンコード両方を
		おこなう
		ただし、このキーワードが使えるのは、MIME を使用するよ
		うにコンパイルされている場合のみに限る。
		デフォルト：0

	GNSPOOL_LINE_LIMIT	gnspool が取寄せる記事の最大行数(E)
		value として 1 以上の数を指定するとそれ以上の行数を持
		つ記事は取寄せない。
		記事の Lines: ヘッダを見るので、Lines: ヘッダを持たな
		い記事はこの指定に関わらず必ず取寄せてしまう。
			！！！警告！！！
		このオプションを指定すると、記事の内容に関わらず取得し
		ないので、有用な記事も取込み損ねる可能性がある。
			！！！警告！！！

		デフォルト：0
