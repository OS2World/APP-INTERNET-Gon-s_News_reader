


GNLOOPS(1)         UNIX-OMRON SOFTWARE tool box-Manual      GNLOOPS(1)



名前
	gnloops - 記事転送ツール

形式
	gnloops [-h hostname] [-V] -d hostname
		-h hostname	gnspool が記事を取寄せたニュースサーバを
				指定する
		-d hostname	記事を転送するニュースサーバを指定する
		-V		バージョンの表示

特長
	gnloops は gnspool で取寄せた記事を他のニュースサーバに転送す
	るツールである。
	ネットワークでつながっていないニュースサーバへ記事を配送したり、
	複数のサーバから記事を取得したり、
	配送されていない経路での記事の配送をおこなうことができる。

機能説明
	gnloops は、gnspool が取寄せた記事を別のサーバに転送する。

	（１）	gnspool が作成した active を参照し、すべてのニュースグ
		ループに対して以下の処理を行う

	（２）	gnspool が取寄せた記事１本１本に対し
		1)記事の Message-ID を探し
		2)ニュースサーバに ihave Message-ID を送信 *1
		3)レスポンスが
			・CONT_XFER なら記事を送信
			・ERR_GOTIT なら次の記事へ

	*1 よって、gnloops を実行するマシンが、転送先のサーバで
	   ”転送許可”と設定されている必要がある。


使用上の留意点
	gnspool が取得する記事のニュースグループや配布範囲には十分注意
	のこと。詳しくは docs.jp/gnloops/readme を参照のこと

	（１）ローカルニュースグループの意図しなかった範囲へ記事の流出
	（２）ローカルディストリビューションの意図しなかった範囲への
		記事の流出
	（３）転送先に存在しないニュースグループ

	転送先のサーバの設定で gnloops を使うマシンからの転送を許可す
	る必要がある。gnloops を実行するマシン（すなわち転送許可の出て
	いるマシン）から転送先のサーバのニュースを gn などのニュースリー
	ダを使って読む場合、ニュースサーバによっては mode reader コマ
	ンドを発行しなければならない場合がある。

動作指定
	gnloops は gn と同じ以下の方法で動作を規定することができる
		(1) 初期化ファイル（$HOME/.gnrc-$NNTPSERVER）
		(2) コマンドラインオプション
		(3) 環境変数
		(4) 初期化ファイル（$HOME/.gnrc）
		(5) コンパイル時に指定された値
	詳細については、gn のマニュアルを参照のこと。

BUGS
	クロスポストされた記事も、クロスポストを考慮せずに転送する。
	よって転送先のニュースサーバで duplicate が発生する場合がある。

	配送経路（Path: ヘッダ）は一切考慮せずに転送する。
	よってすでに通過したニュースサーバへ転送しようとする場合がある。
	この場合確実に ERR_GOTIT になる。
				      
	記事の古さ（Date: ヘッダ）は、一切考慮せずに転送する
	よって転送先のサーバで「そんな古い日付の記事は受け取らない」と
	して展開されない場合がある

	転送先のサーバで expire されたかどうかは一切関知しない。
	転送先のサーバで history からも削除されていた場合、同じ記事が
	転送される可能性がある

	コントロールメッセージは control にポスト（？）されるため、
	control を転送しない場合、記事のキャンセルやニュースグループの
	改廃といった情報が転送されない。ただ、control はすべてのニュー
	スグループに共通のニュースグループ（？）なので、転送は推奨しな
	い。
